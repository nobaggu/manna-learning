# 왜 코사인 유사도를 쓰는가?
- 유클리드 거리도 있음. 그러나 임베딩에선 코사인을 쓰는
이유가 벡터의 길이는 무시하고 방향만 보기에.
- 같은 의미의 짧은 문장과 긴 문장은 벡터 크기가 다를 수 있으나 유클리드 거리로 재면 길이 차이 때문에 멀어보임. 코사인은 방향만 보니까 길이 무관하게 의미 비교 가능
  
# 코사인 유사도는 -1~+1 open ai 임베딩은 실전에서 0~+1사이. 
- sentences = {
    "A": "멘토를 찾고 싶어",
    "B": "조언해줄 사람이 필요해",
    "C": "오늘 점심 뭐 먹지",
}
로 쓸시, a~c가 딱히 의미가 차이 없게 나옴
# 이유
1. 한국어는 임베딩이 좁게 뭉침-open ai 임베딩 모델은 영어 데이터로 많이 학습되었음. so 한국어 문장끼리는 의미와 무관하게 다 비슷하게 클러스터링 되는 경향 있음.
2. 문장이 너무 짧음.
3. 세 문장 모두 한국어 구어체_일상 발화_비슷한 종결어미의 구조적 유사성을 띄기에 모델이 형태적 유사성을 의미적 유사성으로 착각할 만큼 의미 신호가 약함.

# BM25+ Dense hybrid retrieval을 써야 하는 이유는
- Dense(임베딩)-단어 압 겹쳐도 의미 비슷하면 잡아냄.(but 짧은 한국어, 구조 유사 문장에 약함)
- BM25(키워드)-단어 정확히 겹치면 강함, 한국어에도 안정적.(but 동의어, 바꿔말한 표현 못 잡음)-> so hybird형인 RRF쓰는 것
